 ====
 
   1 不仅仅是要完成任务,还需要保证代码的质量。要符合代码规范，包括变量的定义，数字常量化，函数复用性，条件过滤
 校验等等。

   2 做任何任务之前，一定要熟悉业务，了解业务逻辑，弄清背后的需求，考虑各种因素。积极和产品，需求沟通，自己多问几个为什么：为什么会有这样的一个需求？为什么要这样设计？有没有更好的方法去实现等等。

   3 一定要多看文档，不局限于任务的文档，其他相关的文档也可以看看。要多思考，思考背后的本质，深究细节。

   4 在问别人问题之前，自己要思考，要弄清楚自己为什么不明白，要带着问题去问！这样才能提高！不要一遇到问题就去问
   这样会错失自己提高的机会。

   5 任务一旦下达，就一定要去完成，哪怕加班熬通宵！如果有其他因素导致不能及时完成，要及时和boss沟通，说明原因。不能因为个人原因导致项目延期
